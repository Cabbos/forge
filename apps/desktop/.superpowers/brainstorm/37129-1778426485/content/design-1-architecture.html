B + C 混合 — 架构总览

第 1 节：核心抽象

Harness 新架构
EventBus 所有子系统通过事件通信。Agent 发 ToolExecuteRequest，HookEngine 订阅 PreTool/PostTool，SkillLoader 订阅 SessionStart 注入 prompt
Capability trait 统一抽象：fn id() · fn kind() · fn install() · fn enable() · fn disable() · fn configure() · fn events() -> Vec<EventType>
CapabilityRegistry 统一管理所有 Capability：本地扫描 + 远程发现 + 安装/卸载 + 启用/禁用 + 配置。所有能力的数据源
Harness 变薄：初始化 EventBus + Registry，注册内置 Capability，提供 start_session() / process_message() 两个入口

四种 Capability 实例

Skill

SKILL.md 文件。安装到 ~/.ai-studio/skills/。订阅 SessionStart 注入 prompt。可以贡献 Tool 定义。

Hook

脚本或内置函数。订阅 PreTool/PostTool。可修改输入、阻止执行、修改结果。

MCP Server

外部进程。提供额外 Tool。通过 stdio/SSE 通信。可启动/停止。

Tool

内置工具（File/Shell/Search/Web）。也作为 Capability，可 disable。

关键设计决策

  • EventBus 用 tokio::broadcast，多消费者，慢消费者不阻塞
  • Capability trait 用 async_trait，每个能力独立生命周期
  • Registry 数据持久化到 SQLite（已有的 SwiftData → 改用 Rust 的 rusqlite）
  • Harness 不再持有 ToolExecutor，Tool 本身就是 Capability
