第 2 节：数据流 & 调用链

一次用户消息的完整生命周期

Session 生命周期
1. 创建 Session
Frontend createSession() → IPC create_session
→ Harness.start_session(id, working_dir)
→ EventBus.emit(SessionStart)
→ SkillLoader.on_event() 读取启用的 skills
→ Registry.on_event() 读取启用的 tools/hooks/mcp
→ 组装 system_prompt = base + skill_instructions
→ AgentSession 创建完成

2. 用户消息
Frontend sendInput() → IPC send_input
→ AgentSession.send_message(text)
→ Adapter.stream() → SSE 流
→ 解析出 ToolCall 列表

3. 工具执行（每个 ToolCall）
→ EventBus.emit(PreTool { name, input })
→ HookEngine.on_event() 匹配订阅该 tool 的 hooks
→ 逐个执行 pre-hooks（可修改 input / block）
→ PermissionGate.check() 如果有 ask → emit ConfirmAsk 等待
→ Tool.execute(input) → result
→ EventBus.emit(PostTool { name, result })
→ HookEngine.on_event() 执行 post-hooks
→ 结果回填 API history

4. 活跃 Capability 变化（运行时）
→ HubPanel 点击 enable/disable/install
→ IPC 调用 Registry.toggle/enable/install
→ Registry 更新 SQLite + 重新加载
→ EventBus.emit(CapabilityChanged)
→ 后续 session 创建时生效

EventBus 事件类型

SessionStartsession_id, working_dirSkills/Hooks 注入
SessionStopsession_id清理
PreToolsession_id, tool_name, inputHooks 拦截
PostToolsession_id, tool_name, resultHooks 审计
ConfirmAsksession_id, question, kind前端弹窗
CapabilityChangedcapability_id, actionHubPanel 刷新
