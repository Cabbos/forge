第 3 节：迁移路径

从现有代码到新架构，分 3 个 Phase，每步可编译验证

Phase 1：Capability trait + Registry
新增文件
harness/capability.rsCapability trait + CapabilityKind 枚举
harness/registry.rsRegistry struct — 本地扫描 + 安装/卸载/启停 + SQLite 持久化
harness/capabilities/内置 Capability 实现（FileTool, ShellTool, SearchTool, WebTool, BuiltinHooks）
修改文件
harness/mod.rsHarness 初始化 Registry，注册内置 Capability
harness/skills.rsSkillLoader 实现 Capability trait
harness/hooks.rsHookEngine + 每个 Hook 实现 Capability trait
executor/FileTool 等改为 Capability 实现（不删旧文件，先复制再切换引用）
Cargo.toml加 rusqlite
产出
· Capability trait 就绪
· Registry 可扫描本地 Skills/Hooks
· 所有现有 Tool 作为 Capability 注册
· HubPanel 可显示真实数据
Phase 2：EventBus 替换直接调用
新增 / 重写
harness/event_bus.rs重写：tokio::broadcast 替代 Mutex，支持多消费者
agent/session.rssend_message 改造：工具执行改为 EventBus.emit + 等待结果
harness/mod.rsHarness 初始化 EventBus，各子系统订阅事件
产出
· Agent Loop 不再直接调 executor.execute()，改发事件
· HookEngine + PermissionGate 通过事件总线工作
· 前端 FlowEvent 由 EventBus adapter 转发到 Tauri emit
Phase 3：HubPanel 真实数据 + 前端打通
新增 IPC 命令
list_capabilities返回 Registry 中所有能力的 JSON
install_capability从 GitHub/skills.sh 克隆 → Registry 安装
toggle_capabilityenable/disable
configure_capability更新配置 JSON
前端
HubPanel → Skills/MCP/Hooks tabs调用 real IPC，显示真实数据
InputBar修 Enter + 滚动 bug
